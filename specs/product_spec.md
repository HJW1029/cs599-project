# Product Spec: TravelMind Agent

## 1. 背景与问题

用户在陌生城市旅行时，通常需要在地图、点评平台、攻略平台之间反复切换，才能判断附近哪里值得去。传统搜索只返回地点列表，缺少“为什么推荐”和“如何组合成短路线”的解释。

TravelMind Agent 的目标是让用户通过一次地图选点，获得附近可游玩的景点、美食、文化和休闲地点推荐，并给出可解释的推荐理由。

## 2. 用户目标

- 在地图上选择当前位置或目标位置。
- 快速了解周边 1-8km 范围内值得去的地点。
- 根据兴趣偏好获得个性化排序。
- 获得一条简单的半日游路线。

## 3. 核心功能

| 功能 | 说明 |
| --- | --- |
| 地图选点 | 用户点击地图获取经纬度 |
| 周边 POI 查询 | 调用 OpenStreetMap Overpass API 查询附近地点 |
| 智能排序 | 基于距离、类型、评分估计、用户偏好计算推荐分 |
| 推荐解释 | 为每个地点生成自然语言推荐理由 |
| 路线草案 | 根据推荐结果生成半日游停靠顺序 |
| 记忆机制 | 使用 SQLite 保存用户偏好和历史推荐 |
| 可观测性 | 将 Agent 每次执行步骤保存到 traces/ |

## 4. 非功能要求

- 无 API Key 时也能运行。
- Overpass 不可用时使用本地 fallback 数据。
- 推荐逻辑可单元测试。
- 前端适配桌面和移动端。

## 5. MVP 范围

MVP 不实现真实商业评分、登录系统、复杂路线导航和在线支付等功能。评分使用可解释的确定性估计值，重点展示 Agentic workflow、工具调用和推荐解释。
