# CS599 大作业报告提纲

## 一、选题背景与设计思想

- 旅游推荐场景的日常价值。
- 传统地图搜索的不足：信息碎片化、缺少解释、缺少路线组织。
- TravelMind Agent 的设计目标和技术路线。

## 二、Specs 规格文档

- Product Spec: 用户目标、功能边界、MVP 范围。
- Architecture Spec: 系统组件、Agent 状态流、数据流。
- API Spec: 推荐接口、偏好接口、历史接口。

## 三、系统架构与设计

- 总体架构图。
- Agent 工作流状态图。
- POI 工具、记忆模块、推荐排序模块说明。

## 四、关键实现与代码展示

- `TravelMindAgent.recommend` 主循环。
- `fetch_nearby_pois` 工具调用。
- SQLite 记忆机制。
- Trace 输出。
- 前端地图选点。

## 五、测试与评估

- 单元测试：距离计算、排序逻辑、fallback 数据。
- API 测试：健康检查、推荐接口。
- 行为评估：输入固定地点后推荐结果是否相关、理由是否包含距离/评分/偏好。

## 六、系统升级与扩展

- 接入高德/腾讯地图真实评分。
- 接入 DeepSeek API 生成更自然的理由。
- 使用 LangGraph 替换当前轻量状态流。
- 增加用户登录和多用户画像。

## 七、课程总结

- 从“写功能”转向“设计 Agent 工作流”。
- SDD 对降低返工的帮助。
- 工具调用、记忆和可观测性在 Agent 系统中的价值。
